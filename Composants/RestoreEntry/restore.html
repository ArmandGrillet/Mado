The bug : if a file is set as an entry and chrome and is deleted, chrome.fileSystem.isRestorable returns true even if the file is deleted.

To have the bug click on "create a file", its default content is visible in the app (It's "I have to be deleted").

If you relaunch the app, "Is the file still restorable for Chrome?" display "true" and the content of the file is here on launch.
If you remove the file and relaunch the app, "Is the file still restorable for Chrome?" display "true" (it should "false") and the content of the file is not here (because the file is no longer available). The console.log displays nothign even if the file isn't here.
